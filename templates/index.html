System Theory & Algorithms

Detailed explanation of the statistical methods and data simulation used in this project.

1. Sequential Estimation: ARIMA(1,0,1)

The system utilizes an ARIMA(1,0,1) (AutoRegressive Integrated Moving Average) model to establish a dynamic baseline for vehicle speeds. Unlike static thresholds, this model adapts to the momentum and recent history of the traffic flow.

Autoregressive (p=1): Captures the relationship between an observation and its immediately preceding value, essentially modeling the "momentum" of traffic speed.

Integrated (d=0): Indicates the data is stationary; no differencing is required as we model fluctuations around a steady mean.

Moving Average (q=1): Smooths out the series by considering the dependency between an observation and the residual error of the previous step (MA component).

Equation: $y_t = c + \phi_1 y_{t-1} + \theta_1 \epsilon_{t-1} + \epsilon_t$

The difference between the actual speed ($y_t$) and the model's prediction ($\hat{y}_t$) gives us the Residual, which serves as the primary indicator for anomaly detection.

2. Change Detection: CUSUM (Cumulative Sum)

The CUSUM algorithm is a powerful sequential analysis technique designed to detect persistent shifts in the mean. While a single low speed might be a random tap on the brakes, CUSUM accumulates evidence over time, making it highly sensitive to the sustained speed drops characteristic of accidents.

We maintain a running tally, $S_t$, of the cumulative deviations from the expected normal speed $\mu_0$:

Recurrence: $S_t = \max(0, S_{t-1} + (\mu_0 - y_t - \delta))$

  • $\delta$ (Drift): A tolerance parameter that allows for minor fluctuations without increasing the sum.
  • Detection: An alarm is triggered if $S_t$ exceeds a defined threshold (e.g., 5.0). This indicates that the probability of a fundamental change in traffic state is high.

3. Hypothesis Testing: Sequential Probability Ratio Test (SPRT)

The SPRT, developed by Abraham Wald, is an optimal test for deciding between two competing hypotheses as data arrives. It minimizes the amount of data needed to reach a high-confidence decision.

  • $H_0$ (Null Hypothesis): Traffic is normal ($\mu \approx 60$).
  • $H_1$ (Alternative Hypothesis): Accident Condition ($\mu \approx 20$).

We calculate the Log-Likelihood Ratio ($L_t$):

$L_t = L_{t-1} + \log \left( \frac{P(y_t | H_1)}{P(y_t | H_0)} \right)$

As soon as $L_t$ crosses an upper threshold, the system declares an "Accident" state. This approach is much faster and more statistically sound than waiting for a fixed window of time to pass.

4. Data Generation & Simulation

The dashboard simulates a realistic stochastic traffic environment using a generative model:

  • Standard Flow: Initialized with Gaussian noise $N(60, 5)$ to represent fluid highway travel.
  • Accident Injections: Accidents are modeled as sudden shifts in parameters, dropping the mean speed to $N(20, 10)$ and persisting for a random duration (representing response and recovery time).
  • Real-time Updates: Data is pushed via WebSockets to mimic the real-time ingest of GPS pings from a modern telematics platform.
